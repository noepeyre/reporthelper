# Development Workflow

This file describes the expected project behavior and the workflow before publication.

## Goal

The final program must be simple for the end user:

- run `python main.py`;
- press `F8` to configure;
- press `F10` to start or stop;
- press `F9` to quit.

The end user should not have to adjust screen positions. The `layout_reference.json` file must be generated by the developer before publication and shipped with the project.

Several layouts can be shipped in parallel, for example:

- `layout_reference.json`;
- `layout_reference_v2.json`.

In that case, the user chooses the layout in the `F8` configuration window.

## Final App

The `F8` window must contain only:

- target slot, from 1 to 10;
- number of menu options: 4 or 5;
- option to select;
- loop count, with `0 = infinite`;
- speed in ms, with `0 = maximum`;
- random offset in px, with `0 = no offset`;
- layout to use, selected from the available `layout_reference*.json` files.

There must not be a `top` / `bottom` option in the final app.

## Hotkeys

- `F8`: open configuration, only when automation is stopped.
- `F10`: start or stop automation immediately.
- `F9`: quit the program cleanly.

On startup, the app does nothing until a configuration has been saved with `F8`.

## Coordinates

Slots must not be calculated with a simple regular gap.

The visual structure is:

- slots 1 to 5: first group;
- slots 6 to 10: second group;
- between slot 5 and slot 6: larger vertical gap.

Therefore, the final layout must store precise positions for each slot.

The 4-option and 5-option menus must also be separate because their size and positions may change.

The final layout must support:

- menu 4, slot 1: slot button, options 1 to 4, confirmation;
- menu 4, slot 2: slot button, options 1 to 4, confirmation;
- menu 4, slot 10: slot button, options 1 to 4, confirmation;
- menu 5, slot 1: slot button, options 1 to 5, confirmation;
- menu 5, slot 2: slot button, options 1 to 5, confirmation;
- menu 5, slot 10: slot button, options 1 to 5, confirmation.

During automation, the app simply chooses:

```text
if menu_options = 4:
  use layout.menu_4[slot]

if menu_options = 5:
  use layout.menu_5[slot]
```

Then it clicks:

```text
slot button
selected option
confirmation button
```

## Developer Setup Tool

The `setup_layout.py` file is reserved for the developer.

It is used to generate `layout_reference.json`. It is not an end-user feature.

Command:

```powershell
python setup_layout.py
```

At startup, the tool offers two capture orders:

- `current top`;
- `current bottom`.

This choice is only used to simplify the capture order according to the current scoreboard state. It must not be copied into the final app configuration.

## Current Top Order

If setup starts with `current top`, the expected order is:

```text
slot buttons 1-10
slots 1-5 with 5-option menu
confirmations for slots 1-5
slots 6-10 with 4-option menu
confirmations for slots 6-10

slot buttons 1-10
slots 1-5 with 4-option menu
confirmations for slots 1-5
slots 6-10 with 5-option menu
confirmations for slots 6-10
```

## Current Bottom Order

If setup starts with `current bottom`, the expected order is:

```text
slot buttons 1-10
slots 1-5 with 4-option menu
confirmations for slots 1-5
slots 6-10 with 5-option menu
confirmations for slots 6-10

slot buttons 1-10
slots 1-5 with 5-option menu
confirmations for slots 1-5
slots 6-10 with 4-option menu
confirmations for slots 6-10
```

## Publication

Before publication:

1. Activate the virtual environment if used.
2. Install dependencies:

```powershell
python -m pip install -r requirements.txt
```

3. Generate the layout:

```powershell
python setup_layout.py
```

4. Check that `layout_reference.json` exists.
   If several versions exist, also check `layout_reference_v2.json` or any published variants.
5. Test the final app:

```powershell
python main.py
```

6. Test:

- `F8`: configuration;
- `F10`: start / stop;
- `F9`: quit;
- 4-option menu;
- 5-option menu;
- slots 1 to 5;
- slots 6 to 10;
- speed `0`;
- speed with a value in ms;
- offset `0`;
- offset with a value in px;
- every available layout in the list.

7. Publish with:

```text
main.py
requirements.txt
README.md
INSTALL.md
INSTALLfr.md
layout_reference.json
layout_reference_v2.json
reporthelper/
```

`setup_layout.py` can stay private to the developer or be excluded from the final distribution.

## Technical Checks

Useful commands:

```powershell
python -m compileall main.py setup_layout.py reporthelper
python -c "import reporthelper.app; print('app import ok')"
```

The public README must not explain position setup if the script is shipped ready to use.
